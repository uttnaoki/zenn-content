## 退勤前の日報作成

日報の作業内容を書く際、細かい所まで詳しく書くのは面倒なため問題ない程度に簡略化することがあります。
中でも今日やったレビューはいちいち数えてられないため、書かなかったり、人によっては「レビューいっぱい」みたいなカジュアルな書き方をする場合があります。
手軽に今日のレビューを確認できると日報に記載しやすいかなと思いました。


## ブラウザのブックマークレットで今日のレビューを表示

以下をブックマークに登録し、クリックするとjavascriptが実行され、今日の日付を使ったURLへアクセスできます。

```js
javascript:window.location.href='https://github.com/pulls?q=is:pr+reviewed-by:[Githubアカウント]+-author:[Githubアカウント]+updated:%27+new Date().toISOString().slice(0,10);
```

今日（2026/02/01）、私のアカウント（uttnaoki）で実行すると以下のURLにアクセスできます。

```
https://github.com/pulls?q=is:pr+reviewed-by:uttnaoki+-author:uttnaoki+updated:2026-02-01
```

## 感想

レビューを数多く対応した日に日報に書かないのはもったいないため、
ちょっとしたことですがレビュー件数だけでも記載すると作業していることが伝わりますし、
他の方も「もっとレビューしないと」という気持ちになってもらえるかもしれません。

ブックマークレットを使うと今日の日付といった動的な値を使ってURLを叩けるため、
今回の用途以外にも汎用性が高そうです。
